### 4.5 编写FlinkRulInference代码（3.5分钟）

**操作**：IDEA打开FlinkRulInference.scala

**讲解**：

> "FlinkRulInference使用Flink进行实时特征计算，核心是解决三个问题：
>
> **问题1**：高频数据如何计算统计特征？
> 使用1分钟TumblingWindow聚合，计算7个时序特征：var_temp、kurtosis_temp、fft_peak等
>
> **问题2**：高低频数据如何对齐？
> 使用Broadcast State，高频窗口计算完成后广播到所有Task
>
> **问题3**：输出什么格式？
> 22维特征向量，输出到Feature_log供LLM使用"

**展示TumblingWindow聚合**（第309-408行）：

```scala
 // ========== 核心步骤1: 高频数据使用1分钟tumbling window聚合计算7个时序特征 ==========
    val windowSize = Time.minutes(1)

    // 对高频流按设备ID分组，创建1分钟滚动窗口进行聚合
    val timeSeriesFeatureStream: DataStream[TimeSeriesFeatures] = highFreqStream
      // 按设备ID分组，确保同一设备的数据进入同一个窗口
      .keyBy(_.machineId)
      // 使用1分钟处理时间滚动窗口
      .window(TumblingProcessingTimeWindows.of(windowSize))
      .aggregate(
        // 增量聚合函数：逐条累加统计量（count、sum、max等）
        new HighFreqAggregator,
        // 窗口触发后处理：设置machineId并输出结果
        new ProcessWindowFunction[TimeSeriesFeatures, TimeSeriesFeatures, Int, TimeWindow] {
          override def process(key: Int, context: Context,  elements: Iterable[TimeSeriesFeatures], out: Collector[TimeSeriesFeatures] ): Unit = {
            elements.foreach { f =>
              val result = f.copy(machineId = key) // 将聚合结果的machineId设置为当前分组的key
              out.collect(result) // 输出包含7个时序特征的TimeSeriesFeatures对象
            }
          }
        }
      )
      .name("Window-Aggregate-TimeSeriesFeatures")

    val highFreqStateDescriptor =
      new MapStateDescriptor[Int, TimeSeriesFeatures](
        "highFreqFeatures",
        classOf[Int],
        classOf[TimeSeriesFeatures]
      )

    // 将高频窗口结果转为Broadcast Stream，广播到所有并行任务
    val highFreqBroadcast: BroadcastStream[TimeSeriesFeatures] = timeSeriesFeatureStream
        .broadcast(highFreqStateDescriptor) // 广播配置：所有task都能访问这份状态

    val lowFreqConnected = stateStream
      .connect(metricsStream)
      .keyBy(_.machineId, _.machineId)
      .process(new LowFreqCoProcessFunction)
      .name("LowFreq-Merge")
```

**展示Broadcast State合并**（第517-544行）：
```scala
val mergedStream = lowFreqConnected
      .keyBy(_.machineId) // 按设备ID对低频数据分区
      .connect(highFreqBroadcast) // 连接广播流：每个task都能读取所有设备的高频特征
      .process(new MergeHighFreqFunction(highFreqStateDescriptor)) // 自定义处理函数：从Broadcast State读取高频特征并合并
      .name("Merge-HighFreq-Features")

    println(s">>> [输出] 将报警数据写入 Kafka topic: Feature_log")

    val producerProps = new Properties()
    producerProps.setProperty(
      "bootstrap.servers",
      kafkaProps.getProperty("bootstrap.servers")
    )
    producerProps.setProperty("transaction.timeout.ms", "60000")

    val warningSink = new FlinkKafkaProducer[String](
      "Feature_log",
      new org.apache.flink.streaming.connectors.kafka.KafkaSerializationSchema[
        String
      ]() {
        override def serialize(
            element: String,
            timestamp: java.lang.Long
        ): ProducerRecord[Array[Byte], Array[Byte]] = {
          new ProducerRecord("Feature_log", element.getBytes("UTF-8"))
        }
      },
      producerProps,
      FlinkKafkaProducer.Semantic.AT_LEAST_ONCE
    )
```

> **注意**：目前存在一个时序问题——高频窗口是**3分钟**聚合，而低频数据到达更快，导致初期会出现"未找到高频特征，使用默认值0.0"的情况。等第一个窗口计算完成后（约3分钟），数据就会正常合并。
