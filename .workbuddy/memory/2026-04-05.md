## 模拟数据生成脚本重写
- **文件**: `generate_change_data.sh` 重写为**服务器持续运行版**
- **模式**: 无限循环，每3秒输出一条数据（可传参自定义间隔）
- **输出**: `device_state_generated.log` + `sensor_metrics_generated.log`（追加模式，不覆盖）
- **逻辑**: 9台设备随机轮询，状态分布(运行50%/待机47%/离线3%)，累计值递增模拟真实加工
- **用法**: `bash generate_change_data.sh [间隔秒数]` / `nohup ... &` 后台运行
- **原因**: 用户环境 bash 无法本地执行(WSL未安装)，脚本放到 Linux 服务器上运行

  - **加回 feed_rate(进给速度) + spindle_speed(主轴转速)**：用户确认要加回，并给出真实数据范围约束

  - **高频传感器时序数据源**: 新建 `ETL/generate_highfreq_sensor.sh`
    - 目的: 为 Flink 窗口聚合提供 7 个时序特征计算所需的原始高频数据
    - 输出: `highfreq_sensor_generated.log` (JSON, 追加模式)
    - 采样频率: 默认每秒1条(可配置)，9台设备每轮全遍历
    - 10个原始字段: machineId, timestamp, temperature, vibration_x/y/z, spindle_load, feed_rate, spindle_speed, is_running
    - 支持的 Flink 聚合目标:
      * var_temp → 温度方差 (VAR_POP over window)
      * kurtosis_temp → 温度峭度 (UDAF, 冲击检测)
      * fft_peak → 振动频域峰值 (FFT on vibration_x)
      * avg_spindle_load / max_spindle_load → 窗口负载统计
      * running_ratio → 运行占比 (SUM(is_running)/COUNT)
      * avg_feed_rate → 窗口平均进给速度
    - 数据特性: 正态分布(Box-Muller)、缓慢漂移、3%偶发异常尖峰(模拟故障前兆)
    - 与 generate_change_data.sh 配合使用: 一个低频状态+传感器，一个高频时序
  - **FlinkRulInference.scala v2.0 重写**: 三topic消费 + 窗口聚合
    - 新增第3个 Kafka topic: highfreq_sensor (高频时序传感器)
    - HighFreqAggregator (自定义AggregateFunction): 5分钟TumblingWindow内计算7个时序特征
      * var_temp — 总体方差 VAR_POP
      * kurtosis_temp — 超额峰度 (>3=尖峰异常, <3=平坦)
      * fft_peak — 振动频域能量代理(RMS×峰值因子)
      * avg/max_spindle_load — 窗口负载均值/峰值
