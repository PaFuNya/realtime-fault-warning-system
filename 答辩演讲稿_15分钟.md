## 三、核心代码讲解（6分钟）

### 3.1 FlinkRulInference - 实时特征计算引擎（4分钟）

**文件位置**：`src/main/scala/org/example/tasks/FlinkRulInference.scala`

#### 3.1.1 数据流接入（1分钟）

```scala
// 消费Kafka 3个Topic
val deviceStateStream = env.addSource(
  new FlinkKafkaConsumer[String]("device_state", schema, props)
)
val sensorMetricsStream = env.addSource(
  new FlinkKafkaConsumer[String]("sensor_metrics", schema, props)
)
val highFreqStream = env.addSource(
  new FlinkKafkaConsumer[String]("highfreq_sensor", schema, props)  // ★高频流
)
```

**设计要点**：
- 高频流（1秒/条）与低频流（3秒/条）分离处理
- 高频流用于计算**时序统计特征**，低频流用于**设备状态关联**

#### 3.1.2 高频特征计算 - 核心算法（2分钟）

```scala
// ★ 核心：TumblingWindow 1分钟窗口聚合
val highFreqFeatures = highFreqStream
  .keyBy(_.machineId)
  .window(TumblingProcessingTimeWindows.of(Time.minutes(1)))
  .aggregate(
    new HighFreqAggregateFunction(),     // 增量聚合
    new HighFreqWindowProcessFunction()  // 窗口触发时输出
  )
```

**7个时序特征计算逻辑**：

| 特征名 | 计算公式 | 工程意义 |
|--------|----------|----------|
| `var_temp` | VAR_POP = E[X²] - (E[X])² | 温度波动程度，方差大说明温度不稳定 |
| `kurtosis_temp` | μ₄/σ⁴ - 3 | 温度冲击检测，正值=尖峰（有冲击），负值=平坦 |
| `fft_peak` | RMS × 峰值因子 | 振动频域主频率能量，检测机械共振 |
| `avg_spindle_load` | AVG(主轴负载) | 平均负载水平 |
| `max_spindle_load` | MAX(主轴负载) | 峰值负载，检测过载 |
| `running_ratio` | 运行时间/窗口时长 | 设备利用率 |
| `avg_feed_rate` | AVG(进给速度) | 加工效率指标 |

**关键代码片段**（方差计算）：

```scala
// 使用Welford算法在线计算方差（数值稳定）
val meanTemp = sumTemp / count
val variance = meanTempSq - meanTemp * meanTemp  // VAR_POP
```

#### 3.1.3 双流关联与特征合并（1分钟）

```scala
// 低频双流关联：device_state + sensor_metrics
val lowFreqStream = deviceStateStream
  .connect(sensorMetricsStream)
  .keyBy(_.machineId, _.machineId)
  .process(new LowFreqCoProcessFunction())  // 14个低频特征

// ★ 高频特征通过Broadcast State分发到各Task
val broadcastHighFreq = highFreqFeatures
  .broadcast(highFreqStateDescriptor)

// 合并：14低频 + 7高频 = 21维特征向量
val mergedStream = lowFreqStream
  .connect(broadcastHighFreq)
  .process(new MergeHighFreqFunction())
```

**技术亮点**：
- 使用 **Broadcast State** 解决高频窗口特征与低频实时流的时序对齐问题
- 当低频数据到达时，从Broadcast State查询对应设备的最新窗口特征

#### 3.1.4 报警检测与转译

```scala
// 报警字典（错误码 → 中文描述）
val ALARM_DICTIONARY: Map[String, String] = Map(
  "0x01001" -> "OP10 工位：卡料或供料不畅，振动盘频率异常",
  "0x03001" -> "OP30 工位：主轴温度过高，冷却油循环不畅",
  // ... 95个错误码
)

// 检测逻辑
if (record.isAlarm || record.cumulativeAlarms > 0) {
  val alarmMsg = ALARM_DICTIONARY.getOrElse(record.errorCode, "未知报警")
  // 输出到 warning_log
}
```

## 六、Q&A准备

**可能的问题**：

1. **Q: 为什么选择Flink而不是Spark Streaming？**
   - A: Flink是真正的流处理（毫秒级），Spark Streaming是微批（秒级）；Flink的状态管理和窗口语义更完善，适合工业级实时场景。

2. **Q: 为什么选1分钟窗口？**
   - A: 1分钟是权衡结果。足够短以保证实时性，又足够长（60个数据点）保证统计显著性。如果评委质疑，可以说明这是根据实际业务需求调整过的参数。

3. **Q: 如果Flink作业重启，历史数据怎么处理？**
   - A: 使用Kafka的offset回溯 + Flink的checkpoint机制，可以做到exactly-once语义，重启后从断点续传。
