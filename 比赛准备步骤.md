## 二、Kafka Topic 创建（核心步骤）

### 2.1 输入 Topics（Flume → Kafka）

```bash
# Topic 1: sensor_raw (高频传感器原始数据)
kafka-topics.sh --create \
  --topic sensor_raw \
  --partitions 3 \
  --replication-factor 1 \
  --bootstrap-server master:9092

# Topic 2: log_raw (设备日志)
kafka-topics.sh --create \
  --topic log_raw \
  --partitions 3 \
  --replication-factor 1 \
  --bootstrap-server master:9092

# Topic 3: ChangeRecord (变更记录)
kafka-topics.sh --create \
  --topic ChangeRecord \
  --partitions 3 \
  --replication-factor 1 \
  --bootstrap-server master:9092
```

### 2.2 处理 Topics（数据生成脚本 → Kafka）

```bash
# Topic 4: device_state (设备状态 - 低频)
kafka-topics.sh --create \
  --topic device_state \
  --partitions 3 \
  --replication-factor 1 \
  --bootstrap-server master:9092

# Topic 5: sensor_metrics (传感器指标 - 低频)
kafka-topics.sh --create \
  --topic sensor_metrics \
  --partitions 3 \
  --replication-factor 1 \
  --bootstrap-server master:9092

# Topic 6: highfreq_sensor (高频时序传感器 - 1秒/条)
kafka-topics.sh --create \
  --topic highfreq_sensor \
  --partitions 9 \
  --replication-factor 1 \
  --bootstrap-server master:9092
```

### 2.3 输出 Topics（Flink → Kafka）

```bash
# Topic 7: Feature_log (特征+报警输出)
kafka-topics.sh --create \
  --topic Feature_log \
  --partitions 3 \
  --replication-factor 1 \
  --bootstrap-server master:9092

# Topic 8: alert_topic (报警通知)
kafka-topics.sh --create \
  --topic alert_topic \
  --partitions 3 \
  --replication-factor 1 \
  --bootstrap-server master:9092

# Topic 9: device_metrics (设备指标)
kafka-topics.sh --create \
  --topic device_metrics \
  --partitions 3 \
  --replication-factor 1 \
  --bootstrap-server master:9092
```

### 2.4 一键创建所有 Topics（推荐）

```bash
#!/bin/bash
# create_all_topics.sh

BROKER="master:9092"

echo "创建所有 Kafka Topics..."

# 输入 topics
kafka-topics.sh --create --topic sensor_raw --partitions 3 --replication-factor 1 --bootstrap-server $BROKER 2>/dev/null || echo "sensor_raw 已存在"
kafka-topics.sh --create --topic log_raw --partitions 3 --replication-factor 1 --bootstrap-server $BROKER 2>/dev/null || echo "log_raw 已存在"
kafka-topics.sh --create --topic ChangeRecord --partitions 3 --replication-factor 1 --bootstrap-server $BROKER 2>/dev/null || echo "ChangeRecord 已存在"

# 处理 topics
kafka-topics.sh --create --topic device_state --partitions 3 --replication-factor 1 --bootstrap-server $BROKER 2>/dev/null || echo "device_state 已存在"
kafka-topics.sh --create --topic sensor_metrics --partitions 3 --replication-factor 1 --bootstrap-server $BROKER 2>/dev/null || echo "sensor_metrics 已存在"
kafka-topics.sh --create --topic highfreq_sensor --partitions 9 --replication-factor 1 --bootstrap-server $BROKER 2>/dev/null || echo "highfreq_sensor 已存在"

# 输出 topics
kafka-topics.sh --create --topic Feature_log --partitions 3 --replication-factor 1 --bootstrap-server $BROKER 2>/dev/null || echo "Feature_log 已存在"
kafka-topics.sh --create --topic alert_topic --partitions 3 --replication-factor 1 --bootstrap-server $BROKER 2>/dev/null || echo "alert_topic 已存在"
kafka-topics.sh --create --topic device_metrics --partitions 3 --replication-factor 1 --bootstrap-server $BROKER 2>/dev/null || echo "device_metrics 已存在"

echo ""
echo "所有 Topics 创建完成！"
echo "当前 Topics 列表:"
kafka-topics.sh --list --bootstrap-server $BROKER
```

```bash
# 查看每个 topic 的消息数
for topic in sensor_raw log_raw ChangeRecord device_state sensor_metrics highfreq_sensor Feature_log alert_topic; do

  kafka-run-class.sh kafka.tools.GetOffsetShell \
```

## 七、常见问题排查

### 7.1 Kafka 连接失败
```bash
# 检查 Kafka 是否启动
jps | grep Kafka

# 重启 Kafka
kafka-server-stop.sh
kafka-server-start.sh -daemon /opt/module/kafka/config/server.properties
```

### 7.2 Flink 作业失败
```bash
# 查看 Flink 日志
tail -f /opt/module/flink/log/flink-*.log

# 取消失败作业重新提交
bin/flink cancel <job-id>
bin/flink run -d -c org.example.tasks.FlinkRulInference <jar-path>
```

### 7.3 数据生成脚本卡住
```bash
# 检查进程
ps aux | grep generate

# 杀掉卡住的进程
pkill -9 -f generate_highfreq
pkill -9 -f generate_change
pkill -9 -f generate_data

# 重新启动
# (参考 4.1 节)
```
